## 2. Quality Rubric / 质量评分标准

### Scoring Formula / 评分公式

```
Score = SP×0.20 + DK×0.25 + WA×0.15 + RD×0.10 + EQ×0.20 + MC×0.10

SP = System Prompt Depth      (20%)
DK = Domain Knowledge Density (25%)
WA = Workflow Actionability   (15%)
RD = Risk Documentation       (10%)
EQ = Example Quality          (20%)
MC = Metadata Completeness    (10%)
```

### Required 16-Section Structure / 必需的 16 节结构

Every 9.5/10 skill MUST include ALL 16 sections:
<!-- 每个 9.5/10 技能必须包含所有 16 个章节 -->

| # | Section / 章节 | Min Requirement / 最低要求 |
|---|---------------|--------------------------|
| 1 | System Prompt | Role definition + decision framework + thinking patterns + comms style |
| 2 | What This Skill Does | 4+ specific, measurable capabilities |
| 3 | Risk Disclaimer | 5+ domain-specific risks with severity + mitigation |
| 4 | Core Philosophy | Mental model diagram + 3+ guiding principles |
| 5 | Platform Support | Installation for all 7 platforms |
| 6 | Professional Toolkit | 8+ tools with specific purposes and when-to-use |
| 7 | Standards & Reference | Domain frameworks + metrics with target ranges |
| 8 | Standard Workflow | 3+ phases with [✓ Done] and [✗ FAIL] criteria + templates |
| 9 | Scenario Examples | 3+ full conversation flows including edge cases |
| 10 | Common Pitfalls | 5+ named anti-patterns with ❌ BAD / ✅ GOOD + "why it matters" |
| 11 | Integration with Other Skills | 3+ combination workflows with specific outcomes |
| 12 | Scope & Limitations | 3+ "use when" + 3+ "do NOT use when" + alternatives |
| 13 | How to Use This Skill | Quick install command + trigger word list |
| 14 | Quality Verification | Self-checklist + 3 test cases with expected outputs |
| 15 | Version History | 3+ entries with meaningful change descriptions |
| 16 | License & Author | MIT License table + author attribution |

```
Phase 1: Domain Research (4h / AI: 30min)
  → 5-7 core practitioner frameworks

  → Role identity with credentials + specializations

  → Professional Toolkit (10+ tools with specific use cases)
  → Standards & Reference (3+ frameworks + decision trees)
  → Domain Metrics (5+ with formulas and target ranges)
  → Standard Workflow (3 phases with templates + [✓/✗] gates)

Phase 4: Scenarios & Examples (4h / AI: 30min)

  → Each scenario: full User → Expert conversation with artifacts
  → Include calculations, templates, code, checklists as appropriate

Phase 5: Risk & Quality (2h / AI: 15min)
  → Risk Disclaimer (5-7 domain-specific risks + severity)
  → Common Pitfalls (6+ named ❌/✅ anti-patterns)
  → Quality Verification (self-checklist + 3 test cases)
  → Integration (3 skill combinations)
  → Scope & Limitations (clear use/don't-use boundaries)

Phase 6: Metadata & Polish (2h / AI: 15min)
  → Complete all 9 YAML fields
  → Bilingual formatting (Chinese in <!-- --> comments)
  → Version History (3+ entries)
  → License & Author section
```
